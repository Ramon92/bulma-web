## Features
* Based on [@lion/input](https://lion-web-components.netlify.app/?path=/docs/forms-input--main)
* Extra visual elements can be added via `slots`
  * **label:** can also be provided via the `label` attribute, but the `slot` can be used to change the `html` and `CSS` of the label. For example add an `u-sr-only` class to the label to make it visually hidden. A label is always needed for accessibility reasons.
  * **help-text:** a helper text shown below the label to give extra clarification.
  * **before:** can be used to place an icon at the start of the input
  * **after:** can be used to show currency, percentage or an icon
* Delegates attributes like `type`, `disabled`, `placeholder` and `read-only` to the native input.
* __can not yet make use of validation__

### # Read only

`readonly` attribute will be delegated to the native `<input>` to make it read-only. The input will look similar to a normal one, but is not editable and has no shadow.

This field **will still be included** in the parent fieldset or form's `serializedValue`.


<Story name="Read only">
  {html`
  <bw-input readonly .modelValue=${'This is read only'} label="Read only"></bw-input>
  `}
</Story>

```html
<bw-input 
  readonly .modelValue=${'This is read only'} label="Read only">
</bw-input>
```
